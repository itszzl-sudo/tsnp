# tsnp (TypeScript Native Plugin Tool)

**Auto-generate ts-native plugin configuration from Rust crates**

`tsnp` is a command-line tool that analyzes Rust crates for FFI functions and generates TypeScript type definitions and configuration files for the ts-native runtime. This enables TypeScript code to call Rust functions directly without Node.js.

## Installation

```bash
cargo install tsnp
```

Or build from source:

```bash
git clone https://github.com/itszzl-sudo/tsnp.git
cd tsnp
cargo install --path .
```

## Commands

### `cargo tsnp gen <crate-name>`

Generate plugin configuration from a published crates.io crate.

```bash
cargo tsnp gen regex
cargo tsnp gen serde_json
```

**Interactive source selection:**

```
Choose source for FFI function analysis:
[1] Local source code (specify local path)
[2] Download from GitHub (no token required, slower)
[3] Download from GitHub with token (faster via Search API)
```

| Option | Description | Requirements |
|--------|-------------|--------------|
| `[1]` | Scan local directory | Path to crate source |
| `[2]` | Download tarball | None (public repos) |
| `[3]` | Search API | GitHub token |

### `cargo tsnp new <plugin-name>`

Create an empty plugin template for manual configuration.

```bash
cargo tsnp new my-plugin
```

## Output Structure

Both commands generate a `tsnp/<name>/` directory:

```
tsnp/<name>/
├── ts-native.toml    # Plugin configuration
├── index.d.ts        # TypeScript type definitions
└── README.md         # Usage documentation
```

### Generated Files

#### ts-native.toml

```toml
# ts-native Plugin Configuration
# Auto-generated by tsnp for crate 'example'

[package]
name = "tsnp-example"
version = "0.1.0"

[functions]
"js_add" = { impl_name = "js_add" }
"js_multiply" = { impl_name = "js_multiply" }

[link]
lib = "example"
```

#### index.d.ts

```typescript
// TypeScript Type Definitions for ts-native Plugin
// Auto-generated by tsnp for crate 'example'

declare module "tsnp-example" {
    export function js_add(arg0: number, arg1: number): number;
    export function js_multiply(arg0: number, arg1: number): number;
}
```

## Usage Workflow

### 1. Prepare Rust Crate

Ensure your Rust crate exports FFI functions:

```rust
#[no_mangle]
pub extern "C" fn js_add(a: i32, b: i32) -> i32 {
    a + b
}

#[no_mangle]
pub extern "C" fn js_multiply(a: i32, b: i32) -> i32 {
    a * b
}
```

### 2. Generate Plugin

```bash
cargo tsnp gen my-crate
```

### 3. Add Dependency

```toml
# Cargo.toml
[dependencies]
my-crate = "0.1.0"
```

### 4. Use in TypeScript

```typescript
import { js_add, js_multiply } from "tsnp-my-crate";

console.log(js_add(1, 2));       // 3
console.log(js_multiply(3, 4));  // 12
```

## Type Mapping

| Rust Type | TypeScript Type | Notes |
|-----------|-----------------|-------|
| `i8`, `u8`, `i16`, `u16`, `i32`, `u32`, `i64`, `u64`, `isize`, `usize` | `number` | Integer types |
| `f32`, `f64` | `number` | Floating point |
| `*const c_char`, `*mut c_char` | `string` | Null-terminated C string |
| `*const T`, `*mut T` | `number` | Raw pointer (as address) |
| `&T`, `&mut T` | `number` | Reference (as pointer) |
| `()` | `void` | No return value |

### Unsupported Types

These types require manual wrapper functions:

- `Option<T>`, `Result<T, E>` - Use error codes or out parameters
- `Vec<T>`, `[T]` - Pass pointer + length
- `String` - Use `*const c_char`
- Structs with non-POD fields - Create FFI-friendly POD version
- Function pointers / callbacks - Use trampolines

## GitHub Token (Optional)

For faster analysis via GitHub Search API:

```bash
# Set environment variable
export GITHUB_TOKEN="ghp_xxxx"

# Or enter when prompted
cargo tsnp gen some-crate
# Choose [3], paste token
```

Benefits:
- Only fetches files with `#[no_mangle]`
- Faster for large repositories
- Bypasses rate limits

## Integration with ts-native

This tool generates configuration for the [ts-native](https://github.com/your-org/ts-native) runtime:

1. ts-native reads `ts-native.toml` to find function mappings
2. Loads native library (`.dll` / `.so` / `.dylib`)
3. TypeScript calls are translated to FFI calls
4. Type conversions handled automatically

## Troubleshooting

### "Crate not found on crates.io"

- Verify the crate name is correct
- Check if crate is published: `https://crates.io/crates/<name>`

### "GitHub download failed: 404"

- Repository might be private or deleted
- Use option `[1]` with local source instead

### "No FFI functions found"

- Ensure functions are marked with `#[no_mangle]`
- Ensure functions have `extern "C"` ABI
- Functions might be in conditional compilation blocks

### Type inference issues

- Check `ts-native.toml` for commented-out unsupported functions
- Add manual wrappers for complex types
- See README.md in generated directory for guidance

## Architecture

```
main.rs          - CLI entry point, command routing
crates_api.rs    - Fetch crate metadata from crates.io
github_api.rs    - Download/scan Rust source code
parser.rs        - Parse FFI functions, type inference
generator.rs     - Generate toml/d.ts/md files
```
